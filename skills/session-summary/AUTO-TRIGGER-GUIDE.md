# Session-Summary 技能自动触发指南

## 概述
session-summary 技能现在支持根据关键词自动触发。当用户消息中包含特定关键词时，技能会自动启动并生成会话总结文档。

## 自动触发机制

### 1. 触发词列表
技能配置了以下触发词（在 `session-summary.json` 的 `triggers` 字段中）：

**中文触发词**:
- `session-summary` (直接命令)
- `总结会话`
- `生成会话总结`
- `创建会话文档`
- `生成会话`
- `会话总结`
- `技术文档`
- `会议纪要`

**英文触发词**:
- `generate session`
- `session summary`
- `summarize conversation`
- `create documentation`

### 2. 关键词模式匹配
除了精确匹配触发词，技能还支持以下模式匹配：

**中文模式**:
- `(生成|创建|制作|编写).*?(会话|对话|会议|讨论).*?(总结|文档|记录|纪要)`
- `(总结|整理|记录).*?(会话|对话|会议|讨论)`

**英文模式**:
- `(session|conversation|discussion).*?(summary|summarize|document)`
- `(generate|create).*?(document|summary).*?(session|conversation)`

### 3. 触发示例

#### 示例 1: 直接命令
```
用户: session-summary
AI: 正在生成会话总结...
```

#### 示例 2: 中文关键词
```
用户: 帮我把这个对话总结一下
AI: 检测到"总结"关键词，正在生成会话总结文档...
```

#### 示例 3: 英文请求
```
用户: Can you generate a session summary?
AI: Detected "generate session summary", creating session summary...
```

#### 示例 4: 上下文触发
```
用户: 我们刚才讨论了Python装饰器的最佳实践，可以做个记录吗？
AI: 检测到技术讨论结束，正在生成技术文档总结...
```

## 配置说明

### 1. 触发词配置
触发词在 `session-summary.json` 中配置：
```json
"triggers": [
    "session-summary",
    "总结会话",
    "生成会话总结",
    "创建会话文档",
    "生成会话",
    "会话总结",
    "技术文档",
    "会议纪要",
    "generate session",
    "session summary",
    "summarize conversation",
    "create documentation"
]
```

### 2. SKILL.md 配置
技能描述中包含详细的触发条件说明：
```yaml
description: |
  将OpenCode会话内容总结为有趣的Markdown和HTML文档。自动提取代码技巧、干货内容，生成交互式文档。
  支持默认路径(g:/blog)和自定义路径。
  
  触发条件：
  - 用户提到"生成会话"、"总结会话"、"创建会话文档"等关键词
  - 用户请求生成技术文档或会议纪要
  - 用户需要将对话内容整理成结构化文档
  - 用户提到"session-summary"、"总结"、"文档"等关键词
```

## 高级配置

### 1. 添加自定义触发词
要添加自定义触发词，编辑 `session-summary.json`：
```json
"triggers": [
    ...现有触发词...,
    "你的自定义触发词",
    "another-custom-trigger"
]
```

### 2. 调整触发灵敏度
可以通过修改关键词匹配逻辑来调整触发灵敏度：
- **严格模式**: 只匹配精确触发词
- **宽松模式**: 匹配关键词和模式（当前模式）
- **智能模式**: 结合上下文分析（未来功能）

### 3. 上下文感知触发
技能支持上下文感知触发，例如：
- 长时间技术讨论后用户说"整理一下"
- 代码示例展示后用户说"保存这些代码"
- 问题解决步骤说明后用户说"记下来"

## 故障排除

### 问题 1: 技能不触发
**可能原因**:
1. 触发词配置错误
2. 关键词匹配逻辑问题
3. 编码问题

**解决方案**:
1. 检查 `session-summary.json` 中的 `triggers` 字段
2. 运行测试脚本验证匹配逻辑
3. 确保文件使用UTF-8编码

### 问题 2: 误触发
**可能原因**:
1. 触发词太宽泛
2. 模式匹配过于宽松

**解决方案**:
1. 调整触发词列表，移除过于宽泛的词
2. 修改模式匹配正则表达式
3. 添加上下文过滤条件

### 问题 3: 触发但执行失败
**可能原因**:
1. 路径权限问题
2. 模板文件缺失
3. 会话内容无效

**解决方案**:
1. 检查输出路径权限
2. 验证模板文件存在
3. 提供有效的会话内容或session_id
